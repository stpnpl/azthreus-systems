Deep Space Comms

DTN Suite

Delay-Tolerant Networking for the interplanetary age. Our DTN Suite implements the Bundle Protocol standard with the reliability and determinism required for spacecraft operations where round-trip signal times span minutes or hours.

Overview

Built on RFC 5050 (Bundle Protocol), the DTN Suite implements store-and-forward messaging that allows nodes to accumulate data during communication blackouts and forward it opportunistically when contact windows reopen. The C-based ION (Interplanetary Overlay Network) implementation uses zero dynamic memory allocation — a hard requirement for radiation-hardened flight environments.

Key Capabilities

  • Store-and-forward message custody across multi-hop interplanetary paths
  • Deterministic scheduling of contact windows with CFDP (CCSDS File Delivery Protocol) support
  • Zero dynamic memory allocation — all static or pool allocation for flight safety
  • Blackout-resilient operation with automatic retransmission on re-contact
  • Custody transfer for guaranteed delivery across unreliable long-delay links

Cryptography

Quantum-Resistant Cryptographic Core

Future-proof cryptographic infrastructure based on NIST-standardized post-quantum algorithms, protecting mission-critical communications against the threat of large-scale quantum computers.

Overview

Classical cryptographic algorithms — RSA, ECDSA, Diffie-Hellman — are vulnerable to Shor's algorithm running on a sufficiently powerful quantum computer. The Quantum-Resistant Cryptographic Core implements ML-KEM (CRYSTALS-Kyber) for key encapsulation and ML-DSA (CRYSTALS-Dilithium) for digital signatures, both finalized NIST PQC standards. Designed for interstellar communications, telemetry authentication, and command uplink integrity with no managed runtimes and minimal side-channel attack surface.

Key Capabilities

  • ML-KEM-1024 key encapsulation for mission-critical key exchange
  • ML-DSA-87 digital signatures for command authentication and telemetry signing
  • Hybrid classical/PQ modes for transitional interoperability
  • Constant-time implementations to resist timing side-channel attacks
  • HSM-integrated key ceremony support and Shamir Secret Sharing key management

Flight OS

RhAOS

Radiation-Hardened Autonomous Operating System — engineered to survive the cosmic radiation environment of deep space, where a single bit flip can cascade into mission failure.

Overview

Cosmic ray interactions with silicon can flip bits, corrupt memory, and send processors into undefined states. RhAOS addresses this threat at every level: Triple Modular Redundancy (TMR) at the computational unit level, majority-voting logic that resolves disagreements between redundant components, and continuous self-scrubbing memory that corrects single-event upsets (SEUs) before they accumulate into multi-bit errors. The result is an OS that gracefully handles radiation environments that would halt conventional systems.

Key Capabilities

  • Triple Modular Redundancy (TMR) with majority-voting at every safety-critical computation
  • Continuous memory scrubbing to correct single-event upsets from cosmic radiation
  • Provably bounded stack depths — recursion prohibited in flight paths
  • FDIR (Fault Detection, Isolation, and Recovery) subsystem with documented state machine
  • No dynamic memory allocation in flight code paths; all loops have provable termination
  • Hardware Abstraction Layer (HAL) fully isolated from mission application logic

Long-Term Preservation

Humanity Archive Platform

A digital preservation system built on geological timescales — OAIS-compliant infrastructure designed to ensure human knowledge remains readable and intact for 100+ years.

Overview

Inspired by the Svalbard Seed Vault and the GitHub Arctic Code Vault, the Humanity Archive Platform addresses the fundamental archival challenge: formats become obsolete, media degrades, and institutions fail. Our OAIS (ISO 14721)-compliant platform combines automated bitstream preservation, format migration pipelines, and software emulation to ensure that content deposited today remains accessible to readers a century from now — regardless of what changes in the computing landscape.

Key Capabilities

  • OAIS (ISO 14721) reference model compliance for multi-generational preservation
  • Automated bitstream fixity checking with cryptographic checksums and redundancy mapping
  • Format migration pipeline — automated transformation as formats become obsolete
  • Software emulation layer for 100+ year readability without original software
  • Extensive metadata registries capturing provenance, technical constraints, and transformation history
  • ML-DSA-87 + SHA-384 signing for long-term archive integrity verification
